### PART 1: STANDARD VAE TRAINING (02_train_vae.ipynb)

OBJECTIVE: Train a Variational Autoencoder to reconstruct ECG signals while preserving diagnostic features

ARCHITECTURE:
- Input: Single-lead ECG, 2500 samples (10 seconds at 250Hz)
- Encoder: 3 conv layers [1→32→64→64 channels], stride 2, GroupNorm, SiLU activation
- Latent Space: 64 channels × 312 length = 19,968 dimensions
- Compression Factor: 8x (2500 → 312)
- Decoder: 3 transposed conv layers, output range [-1.5, 1.5] mV (clinical range)
- Output: Reconstructed ECG, 2500 samples

TRAINING CONFIGURATION:
- Batch size: 64
- Epochs: 60 (extended training)
- Learning rate: 5e-5
- Loss: MSE + KL divergence + Perceptual loss (using Phase 1 classifier)
- Loss weights: λ_KL = 0.0001, λ_perceptual = 0.8

KEY INNOVATION - PERCEPTUAL LOSS:
- Used the trained Phase 1 AFibResLSTM classifier
- Perceptual loss ensures VAE reconstructions preserve diagnostic features
- Classifier is frozen during VAE training (no gradient updates to classifier)
- Cross-entropy loss between classifier predictions on original vs reconstructed ECGs

RESULTS ACHIEVED:
- Reconstruction MSE: ~0.01-0.02 (very good)
- Latent space is well-organized (no posterior collapse)
- VAE successfully reconstructs ECG waveforms

CLINICAL VALIDATION TEST:
- Ran Phase 1 classifier on both original and VAE-reconstructed ECGs
- Measured "Prediction Agreement" - how often classifier gives same prediction on original vs reconstructed
- Results:
  * Overall Agreement: ~90%+
  * AFib Agreement: High (classifier still detects AFib after reconstruction)
  * Normal Agreement: Slightly lower (room for improvement)
  * Confidence preserved: ~95%+

LATENT SPACE INTERPOLATION TEST:
- Encoded AFib sample → latent space
- Encoded Normal sample → latent space
- Interpolated between them
- Decoded each interpolation step
- Classifier shows smooth transition from AFib→Normal prediction
- This proves latent space captures diagnostic features!

VERDICT: VAE is ready for counterfactual generation

### PART 2: CLASS-CONDITIONAL VAE (02_train_vae_class_conditional.ipynb)

OBJECTIVE: Improve VAE by adding class conditioning to better separate AFib and Normal patterns

MOTIVATION:
- Standard VAE doesn't explicitly know the class label
- Class-conditional VAE can learn class-specific reconstructions
- Better for counterfactual generation: we can explicitly condition on target class

ARCHITECTURE CHANGES:
- Added class embedding layer: nn.Embedding(num_classes=2, embedding_dim=64)
- Encoder: Adds class embedding to latent mu and logvar
- Decoder: Adds class embedding to latent z before decoding
- Conditioning type: "add" (class embedding added to latent)

TRANSFER LEARNING APPROACH:
- Started from pretrained standard VAE weights (60 epochs)
- Only trained new class embedding layers from scratch
- Fine-tuned entire model for 30 additional epochs
- Lower learning rate: 2e-5 (vs 5e-5 for training from scratch)

TRAINING RESULTS:
- Successfully trained class-conditional VAE
- Model learns to condition reconstruction on class label
- Perceptual loss still used to preserve diagnostic features

VALIDATION:
- Classifier agreement test on class-conditional VAE
- Compared original vs reconstructed ECG predictions
- Results show class conditioning helps maintain diagnostic features

NEXT STEPS:
- Use class-conditional VAE for counterfactual generation
- Input AFib ECG, condition on Normal class → get counterfactual Normal ECG
- Input Normal ECG, condition on AFib class → get counterfactual AFib ECG
